## 核心概念

Markdown 是 Agent 友好的记忆接口：人能读，git 能跟踪，搜索工具能索引，AI 能直接引用。

## 适合保存

- 项目背景和决策。
- 常用命令和失败记录。
- 选题卡和资料链接。
- Skill 使用说明。

## 不适合保存

- API Key、token、私钥。
- 需要数据库事务的一致性数据。
- 大量二进制文件。

## 实战案例

把 AI 新闻聚合项目的源、筛选规则、发布流程写进 Markdown，Agent 下次就能接着维护，而不是重新问一遍。
