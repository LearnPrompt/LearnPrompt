## 核心概念

思维蒸馏型 Skill 把一套判断框架写给 Agent，例如产品评审、设计 critique、选题分析。它最容易变成空泛口号。

## 写法

1. 给出明确适用场景。
2. 拆成少量维度。
3. 每个维度要求证据。
4. 输出必须包含取舍和下一步。

## 常见错误

- 维度太多，结果像模板填空。
- 只产出评价，不产出行动。
- 没有和具体材料绑定。

## 实战案例

用 Superpower 和 Office Hour 方法论优化产品时，不能只说“更有产品思维”。要落到用户、使用频率、传播路径、首次成功体验和发布材料。
