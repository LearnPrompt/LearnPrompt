## 核心概念

Skill 是一套可复用的工作说明、脚本、模板和验收标准。它让 Agent 在遇到特定任务时，不必每次从零理解。

## 适合做成 Skill 的任务

- 高频重复。
- 有明确输入输出。
- 有固定检查清单。
- 可以通过脚本或模板提升稳定性。

## 不适合做成 Skill

- 只发生一次的临时任务。
- 需求仍在探索期。
- 依赖大量人工判断且没有稳定步骤。

## 实战案例

“淘金”适合做成 Skill：用户提供榜单或信息源，Skill 抓取、快照、对比、生成日报，并保留限制说明。
