## 适用场景

日报、榜单、资料整理、批量迁移、课程生成，都可以做成流水线型 Skill。

## 设计方法

1. 每一步都落盘。
2. 每一步都有输入和输出。
3. 失败时能从中间继续。
4. 最终报告要写清限制。

## 常见错误

把整条流水线写成一个巨大 prompt。这样失败后无法定位，也无法单独重跑。

## 实战案例

ClawHub 日报流水线：抓 Top100 快照、保存 JSON、读取前一日、计算增量、生成 Markdown、写入收件箱。
