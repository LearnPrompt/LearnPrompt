# vertical-research.md — 纵向研究

核对日期：2026-07-11。

## 核心问题

为什么长任务里最容易坏掉的，不是某个单点能力，而是“探索、计划、实现、验收全都挤在同一条对话里”？

## 从表面现象往下拆

### 表面现象

- 前十轮里明明已经说过“先不要改文件”，后面又开始顺手重构。
- 早期定义过的验收标准，到了后半程被新的猜测覆盖。
- 同一个 bug，越聊越长，后面即使 Claude 还在工作，也越来越像在脏上下文里试错。

### 机制层

根据官方 `best-practices` 与 `how-claude-code-works`，Claude Code 的上下文窗口会累积：

- 消息本身；
- 读过的文件内容；
- 运行命令的输出；
- 载入的 rules、skills、memory。

上下文不是无限缓存。随着会话变长，系统会清旧输出、做摘要压缩；但“压缩过”不等于“没有损失”，
早期细节和边界更容易被弱化。于是任务失稳常见的原因不是“模型突然不会写代码”，而是：

1. 探索结论没有冻结成可复用工件；
2. 第二阶段实现仍然拖着第一阶段的噪声；
3. 验收条件没有单独立成一个机械门。

## 为什么只靠 `continue` 不够

`continue` 的作用是延续 workstream，不是净化 workstream。

- 当你只是中断后回来，且当前任务关系没变，`continue` 很合适。
- 但当同一会话里已经塞满错误猜测、无关 diff、长日志和反复追问时，继续追加消息只会让脏上下文更长。

所以问题不该表述为“我还要不要继续这个 session”，而应该先问：

- 我要保留的是完整历史，还是只保留结论？
- 我要回到原分支继续，还是要开一条新分支验证另一种实现？
- 我的下一阶段需要旧上下文，还是只需要 handoff 和 acceptance？

## 五个操作的真实边界

### `compact`

- 作用：还留在同一 session，只把历史压成摘要，省 context。
- 适用：方向没变，只是对话太长；你仍希望 Claude 继承同一工作流和主要上下文。
- 风险：摘要会压缩细节，所以它只能降低噪声，不会自动纠正错误结论。

### `clear`

- 作用：启动一个空上下文，但旧会话保存在本地，可以之后恢复。
- 适用：你明确判断当前会话已经脏了，想重新给一版更干净、更结构化的 prompt。
- 风险：如果你没有先把探索结论冻结成 handoff，清空之后会重新解释一遍，得不偿失。

### `resume`

- 作用：回到已有 session，继续同一历史。
- 适用：任务被打断、换 terminal、换时间，但历史仍值得保留。
- 风险：resume 只是找回上下文，不是筛掉噪声。

### `fork`

- 作用：复制已有历史到新 session ID，原会话不变。
- 适用：想试另一种实现、另一条修复路线，或者在保留原调查脉络的同时另开分支。
- 风险：如果原历史本身已经很脏，fork 只是把脏历史复制到另一个 session。

### checkpoint / rewind

- 作用：恢复代码、会话或两者；也能做局部 summarize。
- 适用：你想在“同一会话”里快速回退刚才的风险操作，或者把一段旁支讨论压缩掉。
- 风险：官方明确写了它不追踪 bash 命令造成的文件修改，所以它不是 git，也不是部署保险。

## 编辑综合：为什么本文强调 handoff

handoff 不是官方内建命令，而是本文根据官方机制给出的工程做法：把 Stage A 的结果压成一份小而完整的
工件，供 Stage B 在干净上下文直接消费。

一个合格 handoff 至少要回答：

- 症状是什么；
- 证据是什么；
- 允许改哪些文件；
- 怎样验收；
- 哪些风险尚未消除。

没有 acceptance command 的 handoff，本质上只是“看法摘要”，不是可执行契约。第二阶段拿到它，
仍然会落回“看起来修好了”的主观判断。

## 本文 Showcase 的证据边界

- 它证明的是：两阶段、两进程、冻结 handoff 的流程隔离可以被本地脚本确定性复现。
- 它不证明：Claude Code 在所有仓库里都会同样表现，或者某个模型一定更强。
- 它与官方文档的关系是：
  - 官方文档给出会话/上下文/checkpoint 的机制；
  - 本地 Showcase 给出“把机制落成工程操作”的例子。

## 最终结论

对长任务而言，真正重要的不是记住更多命令，而是先判断下一步要保留哪一类状态：

- **保留完整工作流**：`continue` / `resume`
- **保留工作流但压缩噪声**：`compact`
- **丢弃脏上下文，只带走冻结结论**：`clear` 后在新 prompt 中喂 handoff
- **保留原调查，再试另一条实现路线**：`fork`
- **回退刚才那次危险编辑**：checkpoint / rewind

这就是本文教学图和 Showcase 要反复说明的那条线：不是所有“继续”都该继续原上下文，也不是所有“重开”
都该从零开始解释。
