# Brief：记忆层——跨会话保存什么、为什么、何时读取、何时淘汰

## 问题

旧占位稿只有 36 行，把「记忆」简化成偏好/事实/项目上下文三类加四步操作，既没讲清记忆层与当前上下文、任务状态、原始证据的边界，也没讲压缩、可追溯、过期/污染/隐私。更关键的是，它把不同产品的功能混着说，容易让读者以为「模型会自己记住」。

## 目标读者

已经在用 Claude Code 或 Codex、写过 CLAUDE.md/AGENTS.md，但被「换个会话又从头解释一遍」「同一个坑反复踩」坑过、想搞清楚什么该写进跨会话记忆、什么不该的开发者。进阶难度。

## 一句话结果

读完能把一条信息正确归类到当前上下文 / 任务状态 / 持久项目指令 / 自动记忆 / 原始证据五个桶里，按生命周期决定是否持久化，并给记忆设一条召回与淘汰纪律。

## 核心主张

记忆层回答「跨会话、跨进程要记住什么」。它的纪律是：只保存可复用的持久事实与教训；每条记忆带上事实+为什么+怎么用+验证日期+作用域；用索引压缩、用指针指向原始证据；按过期/污染/隐私三条边界淘汰。记忆是「写入磁盘 + 召回时注入」的检索机制，不等于模型自动记住——不读取就零作用。

## 非目标

- 不做 Claude Code 与 Codex 的记忆能力排名或「谁更强」。
- 不把 Claude Code 的 auto memory / CLAUDE.md 与 Codex 的 AGENTS.md / 实验特性 memories 混成同一套术语。
- 不覆盖企业级记忆合规、向量库 RAG、第三方记忆插件（如 mem 类 MCP）的实现细节。
- 不触碰真实密钥、不 push、不部署、不写入任何用户私密历史或 session id。

## 需要的证明

- 一手来源核对 Claude Code 两套记忆系统：CLAUDE.md（你写、全量载入、层级）与 auto memory（模型写、MEMORY.md 索引前 200 行/25KB 载入、topic 文件按需、v2.1.59+、机器本地、每仓一目录）。
- 本地 CLI 核对 Codex：`memories` 为实验特性、`auto_compaction` 稳定、会话 resume/fork；跨会话指导的文档化机制是 AGENTS.md（指令）。
- 一个可复现 Showcase：无记忆时重复犯错 → 写入一条可审计教训 → 独立进程召回并避开 → 过期淘汰 → 有记忆但不召回仍犯错（记忆≠自动记住）。

## 验收条件

- 正文 ≥ 5000 字符，去代码后中文解释 ≥ 1800 字符，H2 ≥ 6。
- 删除 SourceCard，底部保留真实一手来源与橙皮书二手署名。
- 至少一张原创教学图（记忆写入/读取/淘汰生命周期 + 五桶按生命周期分工），asset-ledger 许可闭环。
- Showcase 可复现，raw 日志在工作树外采集、脱敏后入库，无 session id / 绝对路径。
- partial validator 通过，starlight 构建通过（49 页）。writer 阶段保持 partial，不写 quality_score，不写 PASS。

## 与相邻文章的分工

- what-is-harness.mdx（order 1）：五组件总览，状态/记忆是其一。
- instruction-layer.mdx（order 2）：指令层，回答「应该怎样工作」；CLAUDE.md 作为「你写的持久指令」与记忆层交叠，本文划清。
- constraint-layer.mdx（order 3）：约束层，回答「什么绝对不能做」；隐私边界是记忆与约束的交界。
- feedback-loop.mdx（order 4）：反馈层，回路内即时信号；已明确「记忆=跨会话，反馈=即时」。
- 本文（order 5）：记忆层，回答「跨会话记住什么」，与上下文/状态/反馈/编排划清边界。
