## 最小结构

```md
# Skill Name

## When to use
说明触发场景。

## Inputs
列出用户需要提供什么。

## Workflow
按顺序列步骤。

## Verification
列出如何判断完成。

## Safety
列出不能做什么。
```

## 操作步骤

1. 先写触发场景，避免 Skill 被误用。
2. 再写输入和输出。
3. 把工作流拆成 5-9 步。
4. 最后写验收和安全边界。

## 常见错误

只写理念，不写输入输出。Agent 读完仍不知道何时触发、怎么执行。
