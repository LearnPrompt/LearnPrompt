## 核心概念

Agent 最容易失败的地方不是写代码，而是误解任务边界。一个好任务要同时写清楚上下文、目标、限制和验收标准。

## 操作步骤

1. 写出背景：为什么要做、已有系统是什么。
2. 写出目标：完成后用户能看到什么变化。
3. 写出限制：不能改什么、不能接入什么、不能破坏什么。
4. 写出验收：跑什么命令、打开什么页面、对比什么结果。
5. 写出失败处理：如果信息不足，应该先问还是先调查。

## 常见错误

- 只给“优化一下”，没有说明面向谁优化。
- 只给技术方案，没有产品目标。
- 验收标准写成“看起来不错”，无法自动检查。

## 实战案例

“把 LearnPrompt 迁移到 Starlight”不是可执行任务；“在迁移分支中新建 `starlight/`，完成 40 篇首批教程、构建通过、不新增 analytics、不删除旧内容”才是可执行任务。
