## 选择方法

- CLI：适合工程任务、自动化、构建和脚本。
- IDE：适合边读边改、局部理解代码。
- Chrome：适合网页上下文、表单、资料整理。
- 桌面：适合跨 App 的本地协作。

## 操作步骤

1. 先判断任务主要对象是代码、网页、文件还是 App。
2. 代码任务优先 CLI 或 IDE。
3. 网页任务优先 Chrome 或浏览器自动化。
4. 本机 App 任务才考虑桌面控制。

## 常见错误

用一个入口解决所有问题。入口不同，权限、上下文和验收方式都不同。
