## 学习目标

你要完成的不是安装本身，而是跑通“读项目、做小改动、验证、提交”的闭环。

## 操作步骤

1. 准备一个已有 git 项目或新建小项目。
2. 打开 Claude Code，让它先只读盘点目录、运行命令和测试。
3. 要求它提出 3-6 条改动计划。
4. 选择一个最小改动执行。
5. 跑测试或构建，并检查 `git diff`。

## 常见错误

- 一上来让它“重构整个项目”。
- 没有让它先读 README 和 package 文件。
- 没有保留失败输出，导致下一轮重复踩坑。
