# LearnPrompt 黄金样稿工程

本目录保存黄金样稿的研究证据，不是文章草稿回收站。每篇样稿必须留下六类可审计工件：

1. `brief.md`：受众、问题、边界、学习目标和验收条件。
2. `horizontal-research.md`：横向比较官方资料、主题地图和同类教程。
3. `vertical-research.md`：围绕一个关键问题向下追到机制、失败模式和边界。
4. `evidence-ledger.md`：正文中的事实对应到来源、验证日期和置信度。
5. `showcase/`：可复制的提示词、命令、代码、输入与输出。
6. `review.md`：独立审稿、问题修正和最终评分。

## 发布闸门

文章满足以下条件才可标为 `showcase_status: verified`：

- 至少一个一手来源支撑核心事实；会变化的工具信息在发布日期重新验证。
- 至少一个可复现 showcase；真实实验与示意代码必须明确区分。
- 正文能回答“为什么、怎么做、什么时候不该做”，而不只是罗列功能。
- 深度教程至少达到 5,000 个正文字符、去除代码后的 1,800 个中文解释字符和 6 个 H2；这只是防止占位稿混入的下限，不替代独立审稿。
- 独立审稿没有未解决的阻断项。
- Showcase 测试和站点构建都通过。

量化评分见 [quality-rubric.md](./quality-rubric.md)。总分达到 85 分只是必要条件；任一发布闸门失败，分数再高也不能发布。
