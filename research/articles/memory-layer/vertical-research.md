# 纵向研究：从「记忆」这个词追到机制、失败模式与边界

中心问题：跨会话到底该保存什么、为什么、何时读取、何时淘汰？沿表层→机制→失败模式→边界向下追。标注哪些是官方事实、哪些是编辑综合。

## 表层现象

换个会话，Agent 就把上次讲清的项目约定忘了，你又从头解释一遍；或者同一个坑反复踩。直觉解法是「把所有历史都塞回上下文」，但上下文越长遵循度越低，而且每次都重付 token。官方（Claude Code memory 文档）把这层说得很直白：每个会话从一个全新上下文窗口开始，靠两套机制跨会话携带知识——你写的 CLAUDE.md 和模型自己写的 auto memory。

## 机制一：按生命周期分五个桶（编辑综合）

把信息按「活多久 + 谁写谁读」分开，是记忆层的核心心智模型：

1. 当前上下文（working context）：本次对话窗口。生命周期=本会话，/clear 即清、/compact 时被压缩。不是持久记忆。
2. 任务状态（task state）：本任务/本次 run。目标、进度、上次失败证据。生命周期=一个任务，跨轮但任务完成即弃。这是状态层（feedback-loop 已界定），不是记忆层。
3. 持久项目指令（CLAUDE.md / AGENTS.md）：你写的规则。生命周期=跨会话，每次全量载入。属指令层，但因跨会话持久，与记忆层交叠。
4. 自动记忆 / 长期记忆（auto memory）：模型写的 learnings/patterns。生命周期=跨会话，索引载入、细节按需。这是记忆层的核心。
5. 原始证据（raw evidence）：日志、diff、测试报告。用指针引用、不内联进记忆。支撑可追溯。

记忆层管的是 3+4+5 这类跨会话持久信息，尤其 4；与 1（上下文）、2（状态）划清。这个五桶是对官方两套系统的操作化归纳，不是产品官方分类。

## 机制二：写什么、为什么写（官方事实 + 综合）

官方 auto memory：模型不是每次都存，而是判断「未来会话是否有用」才存；存的是 build 命令、调试洞见、架构笔记、代码风格偏好、工作流习惯。官方「何时写 CLAUDE.md」清单更是把「为什么写」讲透：模型第二次犯同样的错、review 抓到本该知道的、你又敲了一遍上次的纠正、新同事需要同样上下文。一句话——把「否则你要反复重讲」的东西写下来。这正是 Showcase 复现的：无记忆时两个独立进程重复同一个错。

## 机制三：何时读取——召回是门控，不是全量倒入（官方事实 + 综合）

官方 auto memory 的读取被压缩成两层：`MEMORY.md` 索引每次载入（前 200 行或 25KB，先到先算），topic 文件（debugging.md 等）不在启动时载入、模型按需用文件工具读。这就是「相关性门控 + 索引压缩」：不是把整个记忆库倒进上下文，而是先看索引、需要时再取细节。Showcase 的 `recall.mjs` 把这条读取纪律做成最小模型：按 trigger 命中当前任务才返回。

## 机制四：何时淘汰——过期 / 污染 / 隐私（官方事实 + 综合）

- 过期：一次性状态（「今天镜像抽风」）不是长期事实，过了有效期必须淘汰，否则下一次进程据此做出已不成立的判断。Showcase 用 `expires` + `--today` 演示 EVICT。
- 污染：把任务状态或一次性状态写成永久记忆，就是污染。官方把 CLAUDE.md 定位为「每个会话都该持有的事实」，多步流程/只在局部生效的东西应下沉到 skill 或路径门控 rule，而不是塞进记忆。矛盾条目还会让模型任选其一。
- 隐私：密钥、session id、绝对路径、用户私密历史绝不写入记忆。官方 auto memory 是机器本地、明文 markdown、可 `/memory` 审计与删除——正因为可被人读到，更不能把敏感信息写进去。约束层负责真正拦截，记忆层负责不主动留存。

## 机制五：可追溯性（官方事实 + 综合）

记忆是压缩后的事实索引，不是原始证据的复制。Anthropic 长任务研究把状态压成结构化 artifact、原始证据留在独立日志/报告。落到记忆层：每条教训带「证据指针」（指向 result.txt / diff / 报告），而不是把整段日志抄进记忆。这样既省上下文，又能回溯原始依据。Showcase 的每条 lesson 都带 `证据指针`。

## 失败模式清单（综合）

1. 全量塞回：把整段聊天当记忆，上下文膨胀、遵循度下降。
2. 污染：一次性状态写成永久事实，误导后续进程。
3. 过期不淘汰：旧事实继续注入，与现状冲突。
4. 隐私泄漏：把密钥/session id/绝对路径写进明文记忆。
5. 错层：把「下一轮要记住」的任务状态当跨会话记忆，或反过来把跨会话事实每轮靠上下文重述。
6. 只写不召回：以为写下来模型就会记住——不读取+注入，记忆零作用。Showcase 阶段 4b 直接复现。

## 边界条件（综合）

- 记忆解决的是「跨会话复用」，不解决工具缺失（能力层）、硬拦截（约束层）、即时信号（反馈层）、调度停止（编排层）。
- 记忆是 context，不是强制：写进去不保证被遵守，硬要求用钩子。
- Claude Code 与 Codex 的记忆机制不同名不同实，不能互相套用术语。
