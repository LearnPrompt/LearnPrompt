## 核心概念

Agent 框架通常要解决三件事：用户从哪里进来、任务在哪里执行、结果发回哪里。Gateway、Node、Channel 可以分别对应入口、执行和渠道。

## 读法

1. Gateway：看它如何接收请求和鉴权。
2. Node：看任务如何被调度、执行和隔离。
3. Channel：看它如何接入 IM、网页或其他客户端。
4. Skill：看可复用能力如何被加载。

## 常见错误

只看模型配置，不看消息链路。真实系统的稳定性往往取决于队列、重试、权限和日志。

## 实战案例

把 Claude Code 接到飞书时，问题可能不在模型，而在进程退出、消息桥接、权限或超时策略。
