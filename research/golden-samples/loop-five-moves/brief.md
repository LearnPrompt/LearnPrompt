# Brief：一个 Loop 的五个动作

## 要解决的问题

很多“自动化”只是把一个提示词定时重跑：它不会发现新任务，不会留下可接力的交接物，也不会根据验收结果决定继续、重试或停机。本文要给出最小闭环，并把每一步做成可观察文件。

## 目标读者

已经会写一次性 Agent 任务，准备构建日报、内容生产、代码维护或资料整理循环的实践者。

## 学习目标

读者能够：

1. 解释 Loop Engineering 与 Harness Engineering 的边界。
2. 用发现、交接、验证、持久化、调度描述一次完整循环。
3. 运行一个不依赖模型的最小 Loop，并查看它留下的 artifact。
4. 判断哪些任务不适合进入循环。

## 中心结论

Loop 不是“无限重试”，而是让任务在时间和多次执行之间保持可判断的状态。最小 Loop 必须知道从哪里拿工作、把什么交给 worker、如何验证、保存什么，以及下一次何时发生。

## 明确不做

- 不把 mock worker 冒充真实 Agent。
- 不讨论某个厂商的定时任务 API。
- 不把所有知识工作都自动化；高风险动作保留人工边界。

## 验收条件

- 五个动作各有可观察输出。
- 测试覆盖成功后持久化以及第二次运行进入 idle。
- 正文解释验证失败、成本和停止条件。
- 独立审稿和站点构建通过。
